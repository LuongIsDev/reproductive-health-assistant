TÀI LIỆU CHI TIẾT HỆ THỐNG TRỢ LÝ ẢO SỨC KHỎE SINH SẢN (REPRODUCTIVE HEALTH ASSISTANT)
================================================================================

1. TỔNG QUAN DỰ ÁN
------------------
- Tên hệ thống: Trợ lý ảo Sức khỏe Sinh sản Multi-Agent.
- Mục tiêu: Cung cấp thông tin giáo dục, tư vấn sức khỏe sinh sản, tình dục và an toàn quan hệ một cách chuyên sâu, chính xác và không phán xét.
- Mô hình hoạt động: Client-Server (Máy khách - Máy chủ).

1.1 KIẾN TRÚC AGENTIC (AGENTIC ARCHITECTURE)
--------------------------------------------
Hệ thống được xây dựng theo chuẩn AI Agent hiện đại, không chỉ là một chatbot thông thường mà là một mạng lưới các tác nhân thông minh:
- Tư duy (Reasoning): Khả năng phân tích câu hỏi để tự đưa ra quyết định điều hướng.
- Sử dụng công cụ (Tool Use): Sử dụng cơ chế Function Calling để kết nối logic giữa các Agent.
- Giao thức MCP (Model Context Protocol): Thiết kế theo hướng module hóa, cho phép mở rộng các Agent như các "công cụ" độc lập, giúp hệ thống linh hoạt và dễ nâng cấp.
- Native Agentic Flow: Sử dụng trực tiếp OpenAI SDK kết hợp FastAPI để tối ưu tốc độ phản hồi và đảm bảo tính ổn định của luồng WebSocket Streaming (thay vì dùng các framework trung gian như LangChain).

2. KIẾN TRÚC HỆ THỐNG (ARCHITECTURE)
------------------------------------
Hệ thống được thiết kế theo kiến trúc Multi-Agent (Đa tác nhân) hiện đại, chia nhỏ kiến thức chuyên môn thành các thực thể độc lập.

A. Mô hình Client-Server:
   - Backend (main.py): Server FastAPI đóng vai trò "đầu não", quản lý logic và các Agent.
   - Frontend (chatbot.py): Giao diện dòng lệnh (CLI) nhẹ, chỉ làm nhiệm vụ giao tiếp với người dùng và gửi dữ liệu lên Server.

B. Thiết kế Multi-Agent:
   Hệ thống gồm 1 Agent Điều phối (Coordinator) và 5 Agent Chuyên gia (Specialists).
   - Agent Điều phối: Sử dụng kỹ thuật "Tool-calling" (Gọi công cụ) của LLM để phân tích ý định người dùng và điều hướng câu hỏi đến Agent phù hợp nhất.
   - Agent Chuyên gia: Mỗi agent có một "System Prompt" (Lời nhắc hệ thống) riêng biệt, chứa kiến thức chuyên sâu về một lĩnh vực cụ thể.

3. CHI TIẾT CÁC THÀNH PHẦN AGENT
--------------------------------
Hệ thống chia làm 5 lĩnh vực chuyên môn:

1. GeneralHealthAgent (Sức khỏe chung):
   - Vai trò: Tư vấn vệ sinh, giải phẫu cơ thể, dậy thì và các vấn đề sức khỏe tổng quát.
   - Nguyên tắc: Cung cấp thông tin nền tảng về cơ thể người.

2. STIAgent (Bệnh lây truyền qua đường tình dục):
   - Vai trò: Chuyên sâu về HIV, lậu, giang mai, sùi mào gà, viêm gan B...
   - Nội dung: Triệu chứng, cách lây nhiễm, phương pháp phòng ngừa (bao cao su, PrEP, PEP) và tầm quan trọng của việc xét nghiệm.

3. ContraceptionAgent (Biện pháp tránh thai):
   - Vai trò: Tư vấn về các phương pháp kế hoạch hóa gia đình.
   - Nội dung: Thuốc tránh thai, đặt vòng, bao cao su, cấy que, tránh thai khẩn cấp, hiệu quả và tác dụng phụ.

4. ReproductiveHealthAgent (Sức khỏe sinh sản):
   - Vai trò: Tư vấn về thai kỳ, sinh sản, chu kỳ kinh nguyệt.
   - Nội dung: Khả năng sinh sản, đa nang buồng trứng (PCOS), mãn kinh, chăm sóc mẹ bầu.

5. SafetyConsentAgent (An toàn & Đồng thuận):
   - Vai trò: Tư vấn về tâm lý, mối quan hệ và sự đồng thuận.
   - Nội dung: Ranh giới cá nhân, nhận biết các dấu hiệu lạm dụng, kỹ năng giao tiếp trong mối quan hệ, tài liệu hỗ trợ khủng hoảng.

4. THÔNG SỐ KỸ THUẬT & CÔNG NGHỆ (TECH STACK)
--------------------------------------------
- Ngôn ngữ lập trình: Python 3.13+
- Backend Framework: FastAPI (Hiệu năng cao, hỗ trợ ASGI & WebSocket).
- Giao thức truyền tải: WebSocket (duy trì kết nối thời gian thực).
- Web Server: Uvicorn.
- Cổng kết nối (Port): 8010.
- Endpoint chính: ws://localhost:8010/ws/chat
- Mô hình ngôn ngữ (LLM): SaoLa4-medium (Cung cấp bởi FPT Cloud).
- Thư viện kết nối LLM: OpenAI Python SDK (Tương thích chuẩn OpenAI).
- Quản lý môi trường: .env (Lưu trữ API Key, Endpoint).
- Giao diện người dùng: Rich (Thư viện Python tạo UI đẹp trên terminal).

5. PHƯƠNG PHÁP & KỸ THUẬT XỬ LÝ
--------------------------------
1. Phương pháp Streaming (Thời gian thực):
   Hệ thống sử dụng WebSocket để truyền tải dữ liệu theo từng "Token". Thay vì đợi model tạo xong toàn bộ câu trả lời, mỗi khi một chữ (token) được tạo ra, nó sẽ được đẩy ngay lập tức về Client để hiển thị, giúp giảm thời gian chờ đợi và tăng trải nghiệm người dùng.

2. Phương pháp Routing (Điều hướng):
   Thay vì dùng một model trả lời tất cả, CoordinatorAgent gửi danh sách "Schema" (mô tả chức năng) của 5 Agent chuyên gia cho LLM. LLM sẽ trả về tên Agent cần dùng dưới dạng một lời gọi hàm (Function call).

2. Quản lý ngữ cảnh (Context Management):
   Hệ thống sử dụng kỹ thuật "State-passing history". Lịch sử trò chuyện được lưu trữ tại Client và gửi lên Server trong mỗi yêu cầu. Điều này giúp hệ thống hiểu được các câu hỏi nối tiếp (ví dụ: "Nó có nguy hiểm không?" sau khi nhắc về một căn bệnh cụ thể).

3. Phương pháp Fallback (Dự phòng):
   Nếu LLM không thể quyết định chọn agent nào, hệ thống tự động chuyển câu hỏi về GeneralHealthAgent để đảm bảo người dùng luôn nhận được phản hồi.

4. Định dạng dữ liệu:
   Dữ liệu trao đổi qua lại dưới dạng JSON theo cấu trúc:
   - Request: { "message": "nội dung", "history": [...] }
   - Response: { "agent_used": "tên agent", "answer": "nội dung trả lời", "question": "câu hỏi gốc" }

6. NGUYÊN TẮC ĐẠO ĐỨC & AN TOÀN
-------------------------------
- Tính ẩn danh: Hệ thống không lưu trữ thông tin cá nhân của người dùng trên server.
- Ngôn ngữ: Sử dụng văn phong y khoa, trung lập, tránh gây sợ hãi hoặc xấu hổ.
- Giới hạn: Luôn có cảnh báo "Thông tin chỉ mang tính giáo dục, không thay thế chẩn đoán của bác sĩ" và khuyến khích người dùng đi khám tại cơ sở y tế uy tín.

8. LUỒNG HOẠT ĐỘNG CHI TIẾT (WORKFLOW)
------------------------------------
Quy trình xử lý một câu hỏi diễn ra qua 5 bước chính:

Bước 1: Kết nối & Gửi (chatbot.py)
   - Client thiết lập kết nối WebSocket bền vững với Server.
   - Người dùng nhập câu hỏi, Client gửi gói tin JSON gồm tin nhắn và lịch sử.

Bước 2: Phân tích Ý định (Routing)
   - CoordinatorAgent nhận tin, dùng LLM để quyết định Agent chuyên gia nhanh chóng.
   - Server gửi tin nhắn loại "info" về Client để thông báo Agent nào sẽ xử lý.

Bước 3: Streaming (Xử lý từng từ)
   - Agent chuyên gia bắt đầu chạy ở chế độ `stream=True`.
   - Mỗi token tạo ra được đóng gói thành tin nhắn loại "token" và gửi qua WebSocket.
   - Client nhận token và cập nhật giao diện `rich.live` ngay lập tức.

Bước 4: Kết thúc
   - Khi hoàn tất, Server gửi tin nhắn loại "done" kèm toàn bộ nội dung đã tạo.
   - Client chốt nội dung cuối cùng và lưu vào lịch sử.

Bước 5: Hiển thị & Cập nhật
   - Client nhận dữ liệu, sử dụng thư viện Rich để hiển thị câu trả lời dưới dạng khung (Panel) và định dạng Markdown đẹp mắt.
   - Lưu câu hỏi và câu trả lời vào lịch sử bộ nhớ tạm để phục vụ cho câu hỏi tiếp theo.

9. CƠ CẤU THƯ MỤC DỰ ÁN
-----------------------
- chatbot.py: Client tương tác người dùng.
- main.py: API Server điều phối.
- agents/: Thư mục chứa logic của 6 tác nhân (base_agent.py, agent_coordinator.py, ...).
- config.py & .env: Cấu hình kết nối API.
- requirements.txt: Danh sách thư viện cần thiết (fastapi, uvicorn, openai, requests, rich, python-dotenv).
